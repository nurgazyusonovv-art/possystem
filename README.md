# Кафе POS & QR меню

Кафе үчүн бирдиктүү система: **QR меню** (кардар), **Касса/POS**, **Ашкана экраны** жана **Админ панел**. Буйрутма кассадан/QR'ден ашканага реалтайм өтөт.

## Технологиялар

- **Next.js 16** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS v4** + өзүнчө дизайн системасы
- **Supabase** — Postgres + Realtime (буйрутмалар заматта жаңыланат)
- **TanStack Query** — маалымат кэши
- **qrcode.react** — QR код генерациясы

## Структура

| Бет | Дарек | Кызматы |
|-----|-------|---------|
| QR меню | `/menu/[token]` | Кардар сканерлеп, меню көрөт, заказ берет |
| Касса | `/pos` | Кассир заказ түзөт, төлөм кабыл алат |
| Ашкана | `/kitchen` | Жаңы заказдар, статус: Даярдоодо → Даяр |
| Админ | `/admin` | Меню, столдор/QR, отчёт |

## Орнотуу

1. **Supabase долбоор түзүңүз** — [supabase.com](https://supabase.com)

2. **Базаны даярдаңыз** — SQL Editor'до ИРЕТИ менен иштетиңиз:
   - `supabase/schema.sql` (таблицалар, триггерлер, realtime)
   - `supabase/seed.sql` (демо тамактар, столдор, кызматкерлер)
   - `supabase/security.sql` (PIN жашыруу, төлөм өзгөртүлгүс, индекстер, Storage)
   - `supabase/auth-rbac.sql` 🔐 **Auth RBAC** (чыныгы Supabase Auth, роль боюнча RLS)

3. **`.env.local` түзүңүз** (`.env.example` карагыла):
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=...
   NEXT_PUBLIC_SITE_URL=http://localhost:3000
   ```

4. **Иштетиңиз:**
   ```bash
   npm run dev
   ```
   → http://localhost:3000

## Буйрутманын агымы (бизнес логика)

```
Жаңы (pending) → Даярдоодо (cooking) → Даяр (ready) → Берилди (served) → Төлөндү (paid)
```

- **QR/Касса** жаңы буйрутма түзөт → `pending`
- **Ашкана** статусту жылдырат → `cooking` → `ready`
- **Касса** тамак берилгенде `served`, төлөм кабыл алганда `paid`
- Буйрутманын суммасы Postgres триггери аркылуу автоматтык эсептелет

## Деплой (продакшнга чыгаруу)

### Vercel (сунушталат)
1. Кодду GitHub'ка жүктөңүз
2. [vercel.com](https://vercel.com) → **Import** → репозиторийди тандаңыз
3. **Environment Variables** кошуңуз:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `NEXT_PUBLIC_SITE_URL` = `https://сиздин-домен.vercel.app` (QR коддору ушуну колдонот!)
4. **Deploy** → даяр

> Деплойдон кийин `NEXT_PUBLIC_SITE_URL`'ди реалдуу доменге коюуну унутпаңыз — QR коддору туура шилтеме берет.

## Коопсуздук абалы

`security.sql` иштетилгенден кийин:
- ✅ Кызматкерлердин **PIN коддору жашырылган** (RPC `staff_login` аркылуу гана текшерилет)
- ✅ Төлөмдөр **өзгөртүлгүс/өчүрүлгүс** (бир заказга бир төлөм)
- ✅ Заказдар **өчүрүлбөйт** (тарых сакталат)
- ✅ Сүрөттөр **Storage CDN'де** (base64 эмес)
- ✅ Өндүрүмдүүлүк индекстери

### Auth RBAC (`auth-rbac.sql` менен активдүү) ✅
- Ар кызматкер **чыныгы Supabase Auth колдонуучусу** (PIN → email/сырсөз)
- **Per-tab сессия** — бир браузерде ар таб өз ролу менен иштейт
- Роль боюнча RLS:
  - 💰 Төлөмдөр — кирген кызматкер гана көрөт/жазат
  - 🍽 Меню/кызматкер башкаруу — **админ гана**
  - 📱 QR кардарлар — меню көрүп, заказ берет (кирүүсүз)
  - 🗑 Өчүрүү — эч кимге (тарых корголгон)
